## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Observer + Polling (Frontend-only)

**Justificación:**
*   **Sin cambios en backend:** Se reutilizan los endpoints existentes (`GET /alquileres`, `GET /mis-alquileres`, `GET /mis-facturas`) para detectar cambios.
*   **Polling cada 30s:** Balance entre actualización en tiempo real y carga del servidor. Escalable a WebSockets en el futuro.
*   **BehaviorSubject:** Propagación reactiva de notificaciones a todos los componentes suscritos.
*   **localStorage:** Persistencia ligera para mantener notificaciones entre recargas.

**Componentes Principales:**

*   **Modelo:**
    *   `AppNotification`: Interfaz con id, type, title, message, icon, iconColor, timestamp, read, routerLink.
    *   `NotificationType`: Tipos literales para categorizar notificaciones.

*   **Servicio (`NotificationService`):**
    *   Polling automático con `timer(0, 30000)`.
    *   Detección de cambios de estado mediante `Map<number, string>` comparando polls sucesivos.
    *   Primer poll: genera resumen (solicitudes pendientes, alquileres activos).
    *   Polls subsiguientes: genera notificaciones individuales por cambio detectado.
    *   API pública: `markAsRead()`, `markAllAsRead()`, `dismissNotification()`, `clearAll()`.
    *   Persistencia en localStorage con clave `maquimu_notifications`.
    *   Auto-start/stop según estado de autenticación.

*   **Componente (`NotificationDropdownComponent`):**
    *   Standalone component reutilizable.
    *   Dropdown dark con animación de entrada (slide-in).
    *   Badge rojo pulsante con contador (máximo "9+").
    *   Lista de notificaciones con icono colorizado, título, mensaje, tiempo relativo.
    *   Barra lateral azul para notificaciones no leídas.
    *   Click outside para cerrar (`@HostListener('document:click')`).

*   **UI Navbar/Sidebar (Rediseño CSS):**
    *   Navbar: fondo `#0f172a`, logo con gradiente azul + box-shadow, role chip con borde, user pill con avatar de iniciales.
    *   Sidebar: fondo `#1e293b`, secciones etiquetadas, icon wrappers 32x32 con colores por categoría, active state con `::before` pseudo-element (barra azul izquierda).
