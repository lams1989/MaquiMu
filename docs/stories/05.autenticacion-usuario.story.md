# Historia #05: Autenticación de Usuario

**Estado:** Ready for Dev
**Fecha:** 2025-11-22
**Origen:** Importada de HISTORIAS_DE_USUARIO.md
**Estimación:** 8 Puntos

---

**Como:** Usuario del sistema (Operario o Cliente)
**Quiero:** Registrarme e iniciar sesión de forma segura con mi email y contraseña.
**Para:** Acceder a las funciones y vistas correspondientes a mi rol.

**Criterios de Aceptación:**
*   Puedo registrarme como nuevo usuario (Cliente auto-registro, Operario creado por admin).
*   Si las credenciales son válidas al hacer login, soy redirigido a mi dashboard (Dashboard de Operario o Portal de Cliente).
*   Si las credenciales son incorrectas, se muestra un mensaje de error.
*   La comunicación está cifrada (HTTPS).
*   Las contraseñas se almacenan hasheadas (BCrypt).
*   El sistema usa JWT para mantener la sesión.

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Alcance:** Full-Stack (Backend + Frontend Web + Móvil) - Autenticación y Registro.

**Modelo de Autenticación: JWT (JSON Web Token)**
*   **Flujo de Login:**
    1. Usuario envía email/password.
    2. Backend valida credenciales contra BD.
    3. Si válido, genera JWT firmado con claims (usuario_id, email, rol).
    4. Frontend/Móvil almacena token (localStorage/SharedPreferences).
    5. Peticiones subsecuentes incluyen token en header `Authorization: Bearer <token>`.
*   **Expiración:** 24 horas (configurable).
*   **Refresh Token:** NO en fase inicial.

**Roles del Sistema:**
*   `OPERARIO`: Empleados de MaquiMu (acceso a gestión de inventario, alquileres, finanzas).
*   `CLIENTE`: Clientes que alquilan (acceso a portal, solicitudes, consultas).

**Backend (Spring Boot):**
*   **Entidad:** `Usuario` (tabla `usuarios` en BD).
    *   Campos: `usuario_id`, `nombre_completo`, `email`, `password_hash`, `rol`, `fecha_creacion`.
*   **Endpoints:**
    *   `POST /api/auth/register`: Registro de nuevos usuarios.
        *   Clientes: auto-registro público.
        *   Operarios: requiere autenticación previa de otro operario (fase futura, por ahora permitir).
    *   `POST /api/auth/login`: Autenticación.
        *   Request: `{email, password}`
        *   Response: `{token, usuario: {id, nombre, email, rol}}`
*   **Seguridad:**
    *   BCrypt para hashear contraseñas.
    *   Spring Security con JWT Filter.
    *   Validaciones: Email único, password mínimo 6 caracteres.

**Frontend (Angular):**
*   **Módulo:** `AuthModule`.
*   **Componentes:**
    *   `LoginComponent`: Formulario de login (basado en `inicio-sesion.html`).
    *   `RegisterComponent`: Formulario de registro.
*   **Servicios:**
    *   `AuthService`: Métodos `login()`, `register()`, `logout()`, `isAuthenticated()`, `getToken()`, `getUserRole()`.
*   **Guards:**
    *   `AuthGuard`: Protege rutas que requieren autenticación.
    *   `RoleGuard`: Protege rutas específicas por rol (OPERARIO vs CLIENTE).
*   **Redirección Post-Login:**
    *   `OPERARIO` → `/admin/dashboard`
    *   `CLIENTE` → `/client/portal`

**Móvil (Android):**
*   **Paquete:** `ui.auth`.
*   **Activities:**
    *   `LoginActivity`: Pantalla de login.
    *   `RegisterActivity`: Pantalla de registro.
*   **Repository:**
    *   `AuthRepository`: Métodos `login()`, `register()`, manejo de token en SharedPreferences.

### Especificaciones Técnicas

**API Endpoints:**
*   `POST /api/auth/register`
    *   Body: `{nombre_completo, email, password, rol}`
    *   Response: `{mensaje: "Usuario registrado exitosamente"}`
*   `POST /api/auth/login`
    *   Body: `{email, password}`
    *   Response: `{token, usuario: {usuario_id, nombre_completo, email, rol}}`

**Base de Datos:**
*   Tabla `usuarios` (ya existe en script SQL).
*   Constraint `UNIQUE` en columna `email`.

**Seguridad:**
*   HTTPS obligatorio en producción.
*   JWT firmado con clave secreta (almacenada en variables de entorno).

## Contexto Técnico (SM)

### Base del Refinamiento
**Análisis arquitectónico utilizado:** SÍ
**Decisiones adoptadas:** JWT, Login + Registro, Roles OPERARIO/CLIENTE, Spring Security.

### Análisis de Impacto
*   **Base de Datos:** Tabla `usuarios` ya existe. No requiere migración adicional.
*   **Backend:** Requiere configurar Spring Security, JWT library (jjwt), BCrypt.
*   **Frontend:** Requiere crear `AuthModule`, guards, interceptores HTTP para token.
*   **Móvil:** Requiere crear Activities de auth, SharedPreferences para token, Retrofit interceptor.
