## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Arquitectura Modular por Características (Feature Modules)
*   **Justificación:** Facilita la escalabilidad, la carga perezosa (Lazy Loading) y la organización del código separando responsabilidades (Admin vs Cliente vs Auth).

**Stack Tecnológico:**
*   **Framework:** Angular 16+
*   **Lenguaje:** TypeScript
*   **Estilos:** SCSS (Sass)
*   **Framework UI:** **Bootstrap 5** (Detectado en maquetación actual)
*   **Iconos:** Bootstrap Icons
*   **Gestión de Estado:** Servicios (RxJS BehaviorSubject) para inicio, NgRx evaluable a futuro si crece la complejidad.

**Estructura de Módulos:**
`src/app`
*   `auth`: Módulo para Login/Registro.
    *   Componentes: `LoginComponent` (inicio-sesion.html).
*   `admin`: Módulo para el Dashboard de Empleados.
    *   Componentes: `DashboardComponent`, `InventoryComponent`, `ClientsComponent`, `RentalsComponent`, `FinanceComponent`.
*   `client`: Módulo para el Portal de Clientes.
    *   Componentes: `PortalComponent`, `RentalRequestComponent`, `ClientRegisterComponent`.
*   `shared`: Componentes reutilizables (Navbar, Sidebar, Cards, Loaders).
*   `core`: Servicios singleton (`AuthService`, `ApiService`), Guards, Interceptores.

### Especificaciones Técnicas

**Migración de Maquetación:**
*   Instalar Bootstrap vía npm: `npm install bootstrap bootstrap-icons`
*   Importar estilos en `styles.scss`:
    ```scss
    @import "bootstrap/scss/bootstrap";
    @import "bootstrap-icons/font/bootstrap-icons";
    // Estilos globales personalizados de dashboard.html
    ```
*   Migrar imágenes a `src/assets/images`.

**Routing:**
*   Implementar Lazy Loading para `admin` y `client`.
*   Ruta por defecto: `/auth/login` o `/client/portal` (según definición de negocio, inicialmente Login).
