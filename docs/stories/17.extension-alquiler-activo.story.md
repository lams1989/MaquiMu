# Historia #17: Extensión de Alquiler Activo

**Estado:** Pendiente 🔲
**Fecha Análisis Arquitectónico:** 2026-02-24
**Origen:** Solicitud Directa (Funcionalidad de Negocio)
**Estimación:** 5 Puntos

---

**Como:** Cliente con un alquiler en estado ACTIVO
**Quiero:** Solicitar una extensión de la fecha de fin de mi alquiler activo.
**Para:** Continuar utilizando la maquinaria alquilada si la necesito por más tiempo, evitando crear una nueva solicitud desde cero.

**Criterios de Aceptación:**

1.  **Botón de extensión en alquiler activo:**
    *   En "Mis Alquileres" (`/client/my-rentals`), los alquileres con estado `ACTIVO` muestran un botón **"Solicitar Extensión"**.
    *   Solo alquileres `ACTIVO` pueden solicitar extensión (no PENDIENTE, APROBADO, etc.).

2.  **Modal de solicitud de extensión:**
    *   Al hacer click en "Solicitar Extensión", se abre un modal que muestra:
        *   Información del alquiler actual (maquinaria, fecha inicio, fecha fin actual, costo actual).
        *   Campo para seleccionar la **nueva fecha de fin** (debe ser posterior a la fecha fin actual).
        *   **Cálculo automático del costo adicional** basado en los días extra × tarifa por día de la maquinaria.
        *   Costo total estimado (costo actual + costo adicional).
    *   Se valida que la maquinaria siga disponible para las fechas de extensión (no tenga otro alquiler APROBADO/ACTIVO que se solape).

3.  **Extensión como solicitud pendiente:**
    *   La extensión se registra como una solicitud con estado `PENDIENTE_EXTENSION`.
    *   El operario recibe una notificación: _"El cliente {nombre} solicita extensión del alquiler #{id}"_.
    *   El operario puede **Aprobar** o **Rechazar** la extensión desde `/admin/rentals`.

4.  **Aprobación de extensión por operario:**
    *   En el módulo de gestión de alquileres (`/admin/rentals`), los alquileres con estado `PENDIENTE_EXTENSION` muestran botones **"Aprobar Extensión"** y **"Rechazar Extensión"**.
    *   Al aprobar: la `fechaFin` del alquiler se actualiza, el `costoTotal` se recalcula, el estado vuelve a `ACTIVO`.
    *   Al rechazar: el estado vuelve a `ACTIVO` (sin cambios en fechas/costo), con motivo opcional.

5.  **Validación de disponibilidad:**
    *   El sistema verifica que la maquinaria no tiene otro alquiler APROBADO/ACTIVO durante el periodo de extensión.
    *   Si no hay disponibilidad, se muestra el mensaje: _"La maquinaria no está disponible para las fechas solicitadas."_

## Análisis Arquitectónico

### Diagnóstico del Código Actual

#### Modelo `Alquiler`
**Archivo:** `dominio/.../alquiler/modelo/Alquiler.java`
```java
public class Alquiler {
    private Long alquilerId;
    private Long clienteId;
    private Long maquinariaId;
    private Long usuarioId;
    private LocalDateTime fechaInicio;
    private LocalDateTime fechaFin;
    private BigDecimal costoTotal;
    private EstadoAlquiler estado;
    private String motivoRechazo;
}
```
**Problema:** No soporta extensiones. No tiene campos para guardar la fecha original ni la nueva fecha solicitada.

#### Enum `EstadoAlquiler`
```java
public enum EstadoAlquiler {
    PENDIENTE, APROBADO, RECHAZADO, ACTIVO, FINALIZADO, CANCELADO
}
```
**Problema:** No tiene estado `PENDIENTE_EXTENSION`.

#### Servicio `CalculadorCostoAlquiler`
- Ya calcula costo basado en fechas y tarifa — se puede reutilizar para calcular el costo adicional.

#### Servicio `ValidadorDisponibilidadMaquinaria`
- Ya valida que no haya alquileres APROBADO/ACTIVO superpuestos — se reutiliza directamente.

#### Frontend — `MyRentalsComponent`
- Lista los alquileres del cliente con filtro por estado y modal de detalle.
- **No tiene** botón de extensión ni modal para solicitar extensión.

#### Frontend — `AdminRentalsComponent`
- Gestiona transiciones (Aprobar, Rechazar, Entregar, Finalizar).
- Necesita agregar lógica para Aprobar/Rechazar extensiones.

---

### Decisiones de Diseño

**Patrón:** Extensión del modelo de dominio existente (no se crea entidad nueva).

**Justificación:**
- La extensión modifica un alquiler existente (no es un alquiler nuevo).
- Se agrega un nuevo estado transitorio `PENDIENTE_EXTENSION` a la máquina de estados.
- Se guardan `fechaFinSolicitada` y `costoAdicional` como campos temporales mientras la solicitud es revisada.

  MODIFY COLUMN estado_alquiler

  NOT NULL DEFAULT 'PENDIENTE';

### Máquina de Estados (Actualizada)

```
                                ┌──────────────────────────┐
                                │                          │
[SOLICITUD] → PENDIENTE → APROBADO → ACTIVO ──→ FINALIZADO
                  ↓                    ↑  ↓          
              RECHAZADO                │  PENDIENTE_EXTENSION
                                       │     ↓          ↓
                                       │   (aprobar)  (rechazar)
                                       │     ↓          ↓
                                       └── ACTIVO    ACTIVO
```

| Estado Actual | Acción | Estado Siguiente |
|---------------|--------|------------------|
| `ACTIVO` | Cliente solicita extensión | `PENDIENTE_EXTENSION` |
| `PENDIENTE_EXTENSION` | Operario aprueba extensión | `ACTIVO` (con fechas y costo actualizados) |
| `PENDIENTE_EXTENSION` | Operario rechaza extensión | `ACTIVO` (sin cambios) |
