## 1. Notificaciones Móvil (desde HU #21 AC1.5)

**Estado actual:** ✅ Implementado

**Tareas:**
- [x] Crear `data/model/NotificationModels.kt` — `AppNotification(id, title, message, type, timestamp, read)`
- [x] Crear `data/repository/NotificationRepository.kt` — Polling cada 60s + detección de cambios de estado
- [x] Agregar panel de notificaciones en `fragment_dashboard.xml`
- [x] Persistencia en SharedPreferences

**Detalle de implementación:**
- `NotificationRepository` detecta cambios de estado en alquileres comparando con estados previos guardados en SharedPreferences
- Detecta nuevas facturas comparando IDs actuales con los previamente conocidos
- Panel de notificaciones integrado en el Dashboard con cards para cada notificación
- Iconos diferenciados por tipo: `ic_construction` para alquileres, `ic_receipt` para facturas
- Polling automático cada 60 segundos mientras el Dashboard está visible (onResume/onPause)
- Máximo 20 notificaciones almacenadas, con opción de limpiar
- Formato de tiempo relativo ("Ahora", "Hace X min", "Hace X h", "Hace X d")
