# Historia #04: Registro de Maquinaria

**Estado:** Ready for Dev
**Fecha:** 2025-11-22
**Origen:** Importada de HISTORIAS_DE_USUARIO.md
**Estimación:** 5 Puntos

---

**Como:** Administrador
**Quiero:** Registrar, visualizar, editar y eliminar maquinaria en el sistema.
**Para:** Mantener el inventario actualizado y gestionar los activos disponibles para alquiler.

**Criterios de Aceptación:**
*   La maquinaria queda registrada con: nombre, marca, modelo, serial (único), tarifas (día/hora) y descripción.
*   El estado inicial de una nueva máquina es `DISPONIBLE`.
*   Se puede visualizar la lista completa de maquinaria con paginación (opcional en fase 1) o lista completa.
*   Se puede editar la información de una máquina existente.
*   Se puede eliminar (o desactivar) una máquina del inventario.
*   Validaciones: Serial único, campos obligatorios no vacíos, tarifas positivas.

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Alcance:** Full-Stack (Backend + Frontend) - CRUD Completo.

**Modelo de Datos (Entidad `Maquinaria`):**
*   `id`: Long (PK, Auto-increment)
*   `nombre`: String (Not Null)
*   `marca`: String
*   `modelo`: String
*   `serial`: String (Unique, Not Null)
*   `estado`: Enum [`DISPONIBLE`, `ALQUILADO`, `MANTENIMIENTO`]
*   `tarifaDia`: BigDecimal (Positive)
*   `tarifaHora`: BigDecimal (Positive)
*   `descripcion`: String (Text)

**Backend (Spring Boot - Hexagonal/Capas):**
*   **Domain:** `Maquinaria` (Model), `MaquinariaRepositoryPort` (Port Out).
*   **Application:** `MaquinariaService` (Lógica de negocio, validaciones).
*   **Infrastructure:**
    *   `MaquinariaController` (Adapter In - REST API).
    *   `JpaMaquinariaRepository` (Adapter Out - MySQL).
    *   `MaquinariaEntity` (JPA Entity si se separa del modelo de dominio, o usar misma clase con anotaciones por simplicidad inicial).

**Frontend (Angular):**
*   **Módulo:** `AdminModule`.
*   **Componentes:**
    *   `InventoryComponent`: Vista principal con tabla de listado.
    *   `MachineModalComponent`: Formulario para Crear/Editar (reutilizable).
*   **Servicios:** `MaquinariaService` (Cliente HTTP).
*   **Modelos:** Interfaz `Maquinaria`.

### Especificaciones Técnicas

**API Endpoints:**
*   `GET /api/maquinarias`: Listar todas.
*   `POST /api/maquinarias`: Crear nueva.
*   `PUT /api/maquinarias/{id}`: Actualizar existente.
*   `DELETE /api/maquinarias/{id}`: Eliminar.

**Base de Datos:**
*   Tabla `maquinaria`.
*   Constraint `UNIQUE` en columna `serial`.
