# Historia #06: Registro de Clientes

**Estado:** Ready for Dev
**Fecha:** 2025-11-22
**Origen:** Importada de HISTORIAS_DE_USUARIO.md
**Estimación:** 3 Puntos

---

**Como:** Agente de Alquiler (Operario)
**Quiero:** Registrar, visualizar, editar y eliminar clientes en el sistema.
**Para:** Mantener un registro preciso para contratos y facturación.

**Criterios de Aceptación:**
*   Los datos del cliente (nombre, identificación, email, teléfono, dirección) se guardan en la base de datos.
*   El sistema no permite registrar un cliente con una identificación (`identificacion`) duplicada.
*   Se puede visualizar la lista completa de clientes.
*   Se puede editar la información de un cliente existente.
*   Se puede eliminar un cliente del sistema.
*   Validaciones: Identificación única, email válido, campos obligatorios no vacíos.

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Alcance:** Full-Stack (Backend + Frontend) - CRUD Completo.

**Modelo de Datos (Entidad `Cliente`):**
*   `cliente_id`: Int (PK, Auto-increment)
*   `nombre_cliente`: String (Not Null)
*   `identificacion`: String (Unique, Not Null)
*   `telefono`: String (Optional)
*   `email`: String (Not Null, Valid Email)
*   `direccion`: String (Optional)
*   `fecha_registro`: Timestamp (Auto-generated)

**Backend (Spring Boot - Hexagonal/Capas):**
*   **Domain:** `Cliente` (Model), `ClienteRepositoryPort` (Port Out).
*   **Application:** `ClienteService` (Lógica de negocio, validaciones).
*   **Infrastructure:**
    *   `ClienteController` (Adapter In - REST API).
    *   `JpaClienteRepository` (Adapter Out - MySQL).
    *   `ClienteEntity` (JPA Entity).

**Frontend (Angular):**
*   **Módulo:** `AdminModule`.
*   **Componentes:**
    *   `ClientListComponent`: Vista principal con tabla de listado.
    *   `ClientModalComponent`: Formulario para Crear/Editar (reutilizable).
*   **Servicios:** `ClienteService` (Cliente HTTP).
*   **Modelos:** Interfaz `Cliente`.

### Especificaciones Técnicas

**API Endpoints:**
*   `GET /api/clientes`: Listar todos.
*   `POST /api/clientes`: Crear nuevo.
*   `PUT /api/clientes/{id}`: Actualizar existente.
*   `DELETE /api/clientes/{id}`: Eliminar.

**Base de Datos:**
*   Tabla `clientes` (ya existe en script SQL).
*   Constraint `UNIQUE` en columna `identificacion`.

**Validaciones:**
*   Identificación única (backend + frontend).
*   Email válido (formato).
*   Campos obligatorios: `nombre_cliente`, `identificacion`, `email`.
