# Historia #07: Gestionar Alquileres (Operario)

**Estado:** Refinado (SM) - Lista para Desarrollo
**Fecha Análisis Arquitectónico:** 2025-11-23
**Fecha Refinamiento:** 2025-11-23
**Origen:** Solicitud Directa (Flujo Crítico)
**Estimación:** 5 Puntos

---

**Como:** Operario
**Quiero:** Gestionar el ciclo de vida de los alquileres (aprobar, entregar, finalizar).
**Para:** Controlar el flujo de maquinaria y permitir la facturación.

**Criterios de Aceptación:**

1.  **Listado de Solicitudes:**
    *   Puedo ver una lista de alquileres filtrada por estado (`PENDIENTE`, `APROBADO`, `ACTIVO`).
    *   Puedo ver el detalle de una solicitud.

2.  **Aprobación/Rechazo:**
    *   Puedo cambiar el estado de una solicitud `PENDIENTE` a `APROBADO` o `RECHAZADO`.
    *   Si rechazo, debo poder ingresar un motivo (opcional).

3.  **Entrega (Activación):**
    *   Puedo registrar la entrega de la maquinaria al cliente.
    *   Esto cambia el estado del alquiler a `ACTIVO`.
    *   La maquinaria cambia su estado a `ALQUILADO`.

4.  **Finalización (Devolución):**
    *   Puedo registrar la devolución de la maquinaria.
    *   Esto cambia el estado del alquiler a `FINALIZADO`.
    *   La maquinaria cambia su estado a `DISPONIBLE`.
    *   Se habilita la generación de factura (HU 09).

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** CQRS (Command Side) + Domain-Driven Design (DDD)

**Justificación:**
*   **Claridad:** Comandos explícitos (`Aprobar`, `Entregar`, `Finalizar`) capturan la intención del negocio mejor que un CRUD genérico.
*   **Consistencia:** Garantiza la integridad transaccional al modificar dos agregados (`Alquiler` y `Maquinaria`) simultáneamente.
*   **Seguridad:** Permite aplicar reglas de validación específicas para cada transición de estado.

**Componentes Principales:**

*   **Aplicación (Comandos):**
    *   `ComandoAprobarAlquiler`: ID + Observaciones.
    *   `ComandoRechazarAlquiler`: ID + Motivo.
    *   `ComandoEntregarAlquiler`: ID (Activa alquiler y actualiza maquinaria).
    *   `ComandoFinalizarAlquiler`: ID (Cierra alquiler y libera maquinaria).

*   **Dominio (Entidades):**
    *   `Alquiler`: Métodos `aprobar()`, `rechazar()`, `activar()`, `finalizar()`.
    *   `Maquinaria`: Métodos `alquilar()`, `devolver()`.

*   **Infraestructura:**
    *   `ComandoControladorAlquiler`: Endpoints `PATCH` específicos para cada acción.

### Especificaciones Técnicas

**Transiciones de Estado:**
1.  `PENDIENTE` -> `APROBADO` (Solo Alquiler).
2.  `PENDIENTE` -> `RECHAZADO` (Solo Alquiler).
3.  `APROBADO` -> `ACTIVO` (Alquiler) + `DISPONIBLE` -> `ALQUILADO` (Maquinaria).
4.  `ACTIVO` -> `FINALIZADO` (Alquiler) + `ALQUILADO` -> `DISPONIBLE` (Maquinaria).

**API Endpoints:**
*   `PATCH /api/maquimu/v1/alquileres/{id}/aprobar`
*   `PATCH /api/maquimu/v1/alquileres/{id}/rechazar`
*   `PATCH /api/maquimu/v1/alquileres/{id}/entregar`
*   `PATCH /api/maquimu/v1/alquileres/{id}/finalizar`

*   **Backend:**

    *   Requiere manejo transaccional estricto.
*   **Frontend:**
    *   Interfaz de gestión para operarios.
    *   Botones de acción condicionales según estado.

### Componentes Técnicos a Crear/Modificar

**1. Módulo Dominio (Lógica de Negocio):**
*   `Alquiler`:
    *   `aprobar(String observaciones)`: Valida estado PENDIENTE -> APROBADO.
    *   `rechazar(String motivo)`: Valida estado PENDIENTE -> RECHAZADO.
    *   `activar()`: Valida estado APROBADO -> ACTIVO.
    *   `finalizar()`: Valida estado ACTIVO -> FINALIZADO.
*   `Maquinaria`:
    *   `alquilar()`: Valida DISPONIBLE -> ALQUILADO.
    *   `devolver()`: Valida ALQUILADO -> DISPONIBLE.

**2. Módulo Aplicación (Comandos):**
*   `ComandoAprobarAlquiler`, `ManejadorAprobarAlquiler`.
