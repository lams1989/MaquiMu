# Historia #09: Generación de Factura

**Estado:** Ready for Dev
**Fecha:** 2025-11-22
**Origen:** Importada de HISTORIAS_DE_USUARIO.md
**Estimación:** 5 Puntos

---

**Como:** Operario (Administrador o Agente)
**Quiero:** Generar facturas para los alquileres.
**Para:** Controlar los pagos y la contabilidad.

**Criterios de Aceptación:**
*   La factura se genera asociada a un alquiler (`alquiler_id`).
*   La factura se crea con un monto, fecha de emisión y un estado por defecto ('PENDIENTE').
*   La factura generada es visible en el módulo financiero y en el portal del cliente.
*   Puedo editar el estado de pago de una factura.
*   Puedo consultar todas las facturas del sistema.

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Alcance:** Full-Stack (Backend + Frontend Web) - CRUD Facturas.

**Modelo de Datos (Entidad `Factura`):**
*   `factura_id`: Int (PK, Auto-increment)
*   `alquiler_id`: Int (FK a alquileres)
*   `fecha_emision`: Date (Not Null)
*   `monto`: Decimal (Not Null)
*   `estado_pago`: Enum [`PENDIENTE`, `PAGADO`, `VENCIDO`]

**Backend (Spring Boot):**
*   **Domain:** `Factura` (Model), `FacturaRepositoryPort`.
*   **Application:** `FacturaService` (Lógica de negocio).
*   **Infrastructure:** `FacturaController`, `JpaFacturaRepository`, `FacturaEntity`.

**Frontend (Angular):**
*   **Módulo:** `AdminModule`.
*   **Componentes:** `InvoiceListComponent`, `InvoiceModalComponent`.

### Especificaciones Técnicas

**API Endpoints:**
*   `GET /api/facturas`: Listar todas.
*   `POST /api/facturas`: Crear nueva factura.
*   `PUT /api/facturas/{id}`: Actualizar estado de pago.
*   `GET /api/facturas/cliente/{clienteId}`: Facturas de un cliente específico.
