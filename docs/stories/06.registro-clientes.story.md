**Como:** Agente de Alquiler (Operario)
**Quiero:** Registrar, visualizar, editar y eliminar clientes en el sistema.
**Para:** Mantener un registro preciso para contratos y facturación.

**Criterios de Aceptación:**
*   Los datos del cliente (nombre, identificación, email, teléfono, dirección) se guardan en la base de datos.
*   El sistema no permite registrar un cliente con una identificación (`identificacion`) duplicada.
*   Se puede visualizar la lista completa de clientes.
*   Se puede editar la información de un cliente existente.
*   Se puede eliminar un cliente del sistema.
*   Validaciones: Identificación única, email válido, campos obligatorios no vacíos.

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Arquitectura Hexagonal Multi-módulo + CQRS + Validación por Capas

**Justificación:**
*   Alineación estricta con la arquitectura definida en HU 01 (multi-módulo Gradle).
*   Aplicación de CQRS: Operaciones de escritura (Crear, Actualizar, Eliminar) vs Operaciones de lectura (Listar, Buscar).
*   Dominio desacoplado de frameworks (Cliente sin dependencias Spring).
*   Validación distribuida en las 3 capas (Dominio, Aplicación, Infraestructura).
*   **Patrón consistente con HU 04 (Maquinaria) y HU 05 (Autenticación).**

**Modelo de Datos (Entidad `Cliente`):**
*   `clienteId`: Long (PK, Auto-increment) - Mapea a `cliente_id` en BD.
*   `nombreCliente`: String (Not Null) - Mapea a `nombre_cliente` en BD.
*   `identificacion`: String (Unique, Not Null).
*   `telefono`: String (Opcional).
*   `email`: String (Not Null, Valid Email).
*   `direccion`: String (Opcional).
*   `fechaRegistro`: LocalDateTime (Auto-generated) - Mapea a `fecha_registro` en BD.

**Estructura de Módulos (Backend):**

#### **1. Módulo Dominio (`dominio/`)**

*   **Modelo:**
    *   `dominio/modelo/Cliente.java` - Entidad de dominio pura (sin JPA).

*   **Puertos (Interfaces):**
    *   `dominio/puerto/dao/ClienteDao.java` - Lectura:
        *   `List<Cliente> listarTodos()`
        *   `Optional<Cliente> buscarPorId(Long id)`
        *   `Optional<Cliente> buscarPorIdentificacion(String identificacion)`
        *   `boolean existePorIdentificacion(String identificacion)`
    *   `dominio/puerto/repositorio/ClienteRepositorio.java` - Escritura:
        *   `Cliente guardar(Cliente cliente)`
        *   `void eliminar(Long id)`

#### **2. Módulo Aplicación (`aplicacion/`)**

*   **CQRS - Comandos (Escritura):**
    *   `aplicacion/comando/ComandoCrearCliente.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorCrearCliente.java`:
        *   Validar identificación única (usando `ClienteDao`).
        *   Validar email válido.
        *   Crear `Cliente` (dominio) con fecha de registro automática.
        *   Persistir (usando `ClienteRepositorio`).
    *   `aplicacion/comando/ComandoActualizarCliente.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorActualizarCliente.java`:
        *   Buscar cliente existente.
        *   Validar identificación única (si cambió).
        *   Validar email válido.
        *   Actualizar campos.
        *   Persistir cambios.
    *   `aplicacion/comando/ComandoEliminarCliente.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorEliminarCliente.java`:
        *   Verificar que existe.
        *   Eliminar físicamente (usando `ClienteRepositorio`).

*   **CQRS - Consultas (Lectura):**
    *   `aplicacion/consulta/ConsultaListarClientes.java` - DTO vacío.
    *   `aplicacion/consulta/manejador/ManejadorListarClientes.java`:
        *   Obtener todos los clientes (usando `ClienteDao`).
        *   Retornar lista.
    *   `aplicacion/consulta/ConsultaBuscarCliente.java` - DTO inmutable.
    *   `aplicacion/consulta/manejador/ManejadorBuscarCliente.java`:
        *   Buscar por ID (usando `ClienteDao`).
        *   Retornar cliente o error si no existe.

#### **3. Módulo Infraestructura (`infraestructura/`)**

*   **Adaptadores de Persistencia:**
    *   `infraestructura/adaptador/entidad/ClienteEntity.java` - Entidad JPA.
    *   `infraestructura/adaptador/dao/JpaClienteDao.java` - Implementa `ClienteDao`.
    *   `infraestructura/adaptador/repositorio/JpaClienteRepositorio.java` - Implementa `ClienteRepositorio`.
    *   `infraestructura/adaptador/repositorio/JpaClienteRepository.java` - Spring Data JPA interface.

*   **Controladores REST (Separados por CQRS):**
    *   `infraestructura/controlador/ComandoControladorCliente.java`:
        *   `POST /api/maquimu/v1/clientes` → `ManejadorCrearCliente`
        *   `PUT /api/maquimu/v1/clientes/{id}` → `ManejadorActualizarCliente`
        *   `DELETE /api/maquimu/v1/clientes/{id}` → `ManejadorEliminarCliente`
    *   `infraestructura/controlador/ConsultaControladorCliente.java`:
        *   `GET /api/maquimu/v1/clientes` → `ManejadorListarClientes`
        *   `GET /api/maquimu/v1/clientes/{id}` → `ManejadorBuscarCliente`

### Especificaciones Técnicas

**API Endpoints:**
*   `GET /api/maquimu/v1/clientes` - Listar todos los clientes.
*   `GET /api/maquimu/v1/clientes/{id}` - Buscar cliente por ID.
*   `POST /api/maquimu/v1/clientes` - Crear nuevo cliente.
    *   Body: `{nombreCliente, identificacion, telefono, email, direccion}`
    *   Response: `{id, nombreCliente, identificacion, email, fechaRegistro, ...}`
*   `PUT /api/maquimu/v1/clientes/{id}` - Actualizar cliente existente.
    *   Body: `{nombreCliente, identificacion, telefono, email, direccion}`
    *   Response: `{id, nombreCliente, identificacion, email, ...}`
*   `DELETE /api/maquimu/v1/clientes/{id}` - Eliminar cliente (eliminación física).

**Base de Datos:**
*   Tabla `clientes` (ya existe en BD).
*   Constraint `UNIQUE` en columna `identificacion`.
*   No requiere migración adicional.

**Validaciones:**
*   Identificación única (dominio + constraint BD).
*   Email válido (formato, dominio).
*   Campos obligatorios: `nombreCliente`, `identificacion`, `email`.
*   Campos opcionales: `telefono`, `direccion`.
*   Fecha de registro automática (generada en creación).

**Frontend (Angular):**
*   **Módulo:** `AdminModule`.
*   **Componentes:**
    *   `ClientListComponent`: Vista principal con tabla de listado.
    *   `ClientModalComponent`: Formulario para Crear/Editar (reutilizable).
*   **Servicios:** `ClienteService` (consume endpoints `/clientes`).
*   **Modelos:** Interfaz `Cliente`.
