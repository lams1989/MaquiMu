**Como:** Usuario del sistema (Operario o Cliente)
**Quiero:** Registrarme e iniciar sesión de forma segura con mi email y contraseña.
**Para:** Acceder a las funciones y vistas correspondientes a mi rol.

**Criterios de Aceptación:**
*   Puedo registrarme como nuevo usuario (Cliente auto-registro, Operario creado por admin).
*   Si las credenciales son válidas al hacer login, soy redirigido a mi dashboard (Dashboard de Operario o Portal de Cliente).
*   Si las credenciales son incorrectas, se muestra un mensaje de error.
*   La comunicación está cifrada (HTTPS).
*   Las contraseñas se almacenan hasheadas (BCrypt).
*   El sistema usa JWT para mantener la sesión.

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Arquitectura Hexagonal Multi-módulo + CQRS + Seguridad por Capas

**Justificación:**
*   Alineación estricta con la arquitectura definida en HU 01 (multi-módulo Gradle).
*   Aplicación de CQRS: Registro (Comando) vs Login (Consulta).
*   Dominio desacoplado de frameworks (JWT sin dependencias Spring).
*   Seguridad distribuida en las 3 capas (Dominio, Aplicación, Infraestructura).

**Modelo de Autenticación: JWT (JSON Web Token)**
*   **Flujo de Registro (Comando):**
    1. Usuario envía datos de registro.
    2. `ManejadorRegistrarUsuario` valida email único.
    3. Hashea password con BCrypt.
    4. Persiste usuario usando `UsuarioRepositorio`.
    5. Retorna confirmación.

*   **Flujo de Login (Consulta):**
    1. Usuario envía email/password.
    2. `ManejadorAutenticarUsuario` busca usuario por email.
    3. Valida password con BCrypt.
    4. Genera JWT con claims (usuarioId, email, rol).
    5. Retorna token + datos de usuario.

*   **Expiración:** 24 horas (configurable).
*   **Refresh Token:** NO en Fase 1 (implementación futura).

**Roles del Sistema:**
*   `OPERARIO`: Empleados de MaquiMu (acceso a gestión de inventario, alquileres, finanzas).
*   `CLIENTE`: Clientes que alquilan (acceso a portal, solicitudes, consultas).

**Estructura de Módulos (Backend):**

#### **1. Módulo Dominio (`dominio/`)**

*   **Modelo:**
    *   `dominio/modelo/Usuario.java` - Entidad de dominio pura (sin JPA).
    *   `dominio/modelo/RolUsuario.java` - Enum: `OPERARIO`, `CLIENTE`.

*   **Puertos (Interfaces):**
    *   `dominio/puerto/dao/UsuarioDao.java` - Lectura:
        *   `Optional<Usuario> buscarPorEmail(String email)`
        *   `Optional<Usuario> buscarPorId(Long id)`
        *   `boolean existePorEmail(String email)`
    *   `dominio/puerto/repositorio/UsuarioRepositorio.java` - Escritura:
        *   `Usuario guardar(Usuario usuario)`
        *   `void eliminar(Long id)`

#### **2. Módulo Aplicación (`aplicacion/`)**

*   **CQRS - Comandos (Escritura):**
    *   `aplicacion/comando/ComandoRegistrarUsuario.java` - DTO inmutable.
    *   `aplicacion/comando/fabrica/FabricaUsuario.java` - Fábrica para construcción de Usuario.
    *   `aplicacion/comando/manejador/ManejadorRegistrarUsuario.java`:
        *   Validar email único (usando `UsuarioDao`).
        *   Hashear password (BCrypt).
        *   Usar `FabricaUsuario` para crear `Usuario` (dominio).
        *   Persistir (usando `UsuarioRepositorio`).

*   **CQRS - Consultas (Lectura):**
    *   `aplicacion/consulta/ConsultaAutenticarUsuario.java` - DTO inmutable.
    *   `aplicacion/consulta/manejador/ManejadorAutenticarUsuario.java`:
        *   Buscar usuario por email (usando `UsuarioDao`).
        *   Validar password (BCrypt).
        *   Generar JWT (usando `GeneradorJwt`).
        *   Retornar `RespuestaAutenticacion`.

*   **Servicios de Dominio:**
    *   `aplicacion/servicio/GeneradorJwt.java`:
        *   `String generarToken(Usuario usuario)`
        *   `Claims validarToken(String token)`
        *   `Long extraerUsuarioId(String token)`
        *   `RolUsuario extraerRol(String token)`

#### **3. Módulo Infraestructura (`infraestructura/`)**

*   **Adaptadores de Persistencia:**
    *   `infraestructura/adaptador/entidad/UsuarioEntity.java` - Entidad JPA.
    *   `infraestructura/adaptador/dao/JpaUsuarioDao.java` - Implementa `UsuarioDao`.
    *   `infraestructura/adaptador/repositorio/JpaUsuarioRepositorio.java` - Implementa `UsuarioRepositorio`.
    *   `infraestructura/adaptador/repositorio/JpaUsuarioRepository.java` - Spring Data JPA interface.

*   **Controladores REST:**
    *   `infraestructura/controlador/AuthController.java`:
        *   `POST /api/maquimu/v1/auth/register` → `ManejadorRegistrarUsuario`
        *   `POST /api/maquimu/v1/auth/login` → `ManejadorAutenticarUsuario`

*   **Configuración de Seguridad:**
    *   `infraestructura/configuracion/SeguridadConfig.java` - Extender desde HU 01:
        *   Rutas públicas: `/api/maquimu/v1/auth/**`
        *   Registrar `JwtAuthenticationFilter`.
    *   `infraestructura/configuracion/JwtAuthenticationFilter.java`:
        *   Interceptar peticiones HTTP.
        *   Validar token JWT.
        *   Establecer `SecurityContext`.
    *   `infraestructura/configuracion/AuthEntryPoint.java` - Manejo de errores 401.

### Especificaciones Técnicas

**API Endpoints:**
*   `POST /api/maquimu/v1/auth/register`
    *   Body: `{nombreCompleto, email, password, rol}`
    *   Response: `{mensaje: "Usuario registrado exitosamente"}`
*   `POST /api/maquimu/v1/auth/login`
    *   Body: `{email, password}`
    *   Response: `{token, usuario: {usuarioId, nombreCompleto, email, rol}}`

**Base de Datos:**
*   Tabla `usuarios` (ya existe desde `V1__initial_schema.sql` con `rol` como `ENUM('OPERARIO', 'CLIENTE')`).
*   **Migración `V2__update_rol_enum.sql`:** Script existente para actualizar datos antiguos (`'admin'`, `'agente'`) a los nuevos valores si fuera necesario, no para cambio de esquema.

**Seguridad:**
*   BCrypt para hashear contraseñas.
*   JWT firmado con clave secreta (variable de entorno: `JWT_SECRET`).
*   HTTPS obligatorio en producción.
*   Expiración de token: 24 horas.

**Frontend (Angular):**
*   **Módulo:** `AuthModule`.
*   **Componentes:** `LoginComponent`, `RegisterComponent`.
*   **Servicios:** `AuthService` (consume endpoints `/auth/*`).
*   **Guards:** `AuthGuard`, `RoleGuard`.
*   **Interceptor:** `AuthInterceptor` (añade header `Authorization: Bearer <token>`).
*   **Redirección Post-Login:**
    *   `OPERARIO` → `/admin/dashboard`
    *   `CLIENTE` → `/client/portal`

**Móvil (Android):**
*   **Activities:** `LoginActivity`, `RegisterActivity`.
*   **Repository:** `AuthRepository` (manejo de token en SharedPreferences).
