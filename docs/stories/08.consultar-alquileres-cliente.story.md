---

**Como:** Cliente
**Quiero:** Consultar mi historial de alquileres (web y móvil).
**Para:** Tener trazabilidad de mis servicios contratados.

**Criterios de Aceptación:**
*   Veo una lista de mis alquileres (activos e historial).
*   Puedo ver los detalles de un alquiler específico.
*   La información incluye: maquinaria, fechas, costo, estado.

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Arquitectura Hexagonal Multi-módulo + CQRS (Solo Consultas)

**Justificación:**
*   **Reutilización:** Aprovecha el modelo `Alquiler` y el puerto `AlquilerDao` definidos en HU 07.
*   **CQRS:** Implementa solo la parte de **Consultas (Queries)** para obtener datos.
*   **Seguridad:** Filtrado estricto por el `clienteId` asociado al usuario autenticado (JWT).

**Componentes Reutilizados (de HU 07):**
*   **Dominio:** `Alquiler` (Modelo), `AlquilerDao` (Puerto).
*   **Infraestructura:** `JpaAlquilerDao`, `JpaAlquilerRepository`, `ConsultaControladorAlquiler`.

**Nuevos Componentes (Backend):**
*   **Aplicación (Consultas):**
    *   `ConsultaMisAlquileres`: DTO para solicitar el historial.
    *   `ManejadorMisAlquileres`: Lógica para recuperar alquileres del cliente.
    *   `ConsultaDetalleAlquiler`: DTO para ver un alquiler específico.
    *   `ManejadorDetalleAlquiler`: Lógica para buscar por ID y validar propiedad (que pertenezca al cliente).

**Frontend (Angular + Móvil):**
*   **Vistas:** Listado de alquileres (Historial) y Detalle de alquiler.
*   **Servicios:** Actualizar `AlquilerService` con métodos `getMisAlquileres()` y `getAlquilerById()`.

## Contexto Técnico (SM)

### Base del Refinamiento
**Análisis arquitectónico utilizado:** SÍ
**Decisiones adoptadas:** CQRS para consultas, reutilización de DAO, filtrado por seguridad.

### Análisis de Impacto
*   **Backend:**
    *   No requiere migración de BD.
    *   Requiere modificar `AlquilerDao` para añadir método `buscarPorCliente`.
    *   Requiere actualizar `ConsultaControladorAlquiler`.
*   **Frontend:**
    *   Nuevas pantallas en módulo `ClientModule`.
    *   Integración con API de consultas.
