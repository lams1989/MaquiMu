**Como:** Administrador
**Quiero:** Registrar, visualizar, editar y eliminar maquinaria en el sistema.
**Para:** Mantener el inventario actualizado y gestionar los activos disponibles para alquiler.

**Criterios de Aceptación:**
*   La maquinaria queda registrada con: nombre, marca, modelo, serial (único), tarifas (día/hora) y descripción.
*   El estado inicial de una nueva máquina es `DISPONIBLE`.
*   Se puede visualizar la lista completa de maquinaria con paginación (opcional en fase 1) o lista completa.
*   Se puede editar la información de una máquina existente.
*   Se puede eliminar (o desactivar) una máquina del inventario.
*   Validaciones: Serial único, campos obligatorios no vacíos, tarifas positivas.

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** Arquitectura Hexagonal Multi-módulo + CQRS + Validación por Capas

**Justificación:**
*   Alineación estricta con la arquitectura definida en HU 01 (multi-módulo Gradle).
*   Aplicación de CQRS: Operaciones de escritura (Crear, Actualizar, Eliminar) vs Operaciones de lectura (Listar, Buscar).
*   Dominio desacoplado de frameworks (Maquinaria sin dependencias Spring).
*   Validación distribuida en las 3 capas (Dominio, Aplicación, Infraestructura).
*   Patrón consistente con HU 05 (Autenticación).

**Modelo de Datos (Entidad `Maquinaria`):**
*   `maquinariaId`: Long (PK, Auto-increment) - Mapea a `maquinaria_id` en BD.
*   `nombreEquipo`: String (Not Null) - Mapea a `nombre_equipo` en BD.
*   `marca`: String (Not Null).
*   `modelo`: String (Not Null).
*   `serial`: String (Unique, Not Null).
*   `estado`: Enum [`DISPONIBLE`, `ALQUILADO`, `EN_MANTENIMIENTO`] - Mapea a `('disponible', 'alquilado', 'en_mantenimiento')` en BD.
*   `tarifaPorDia`: BigDecimal (Positive) - Mapea a `tarifa_por_dia` en BD.
*   `tarifaPorHora`: BigDecimal (Positive) - Mapea a `tarifa_por_hora` en BD.
*   `descripcion`: String (Text, Opcional).

**Estructura de Módulos (Backend):**

#### **1. Módulo Dominio (`dominio/`)**

*   **Modelo:**
    *   `dominio/modelo/Maquinaria.java` - Entidad de dominio pura (sin JPA).
    *   `dominio/modelo/EstadoMaquinaria.java` - Enum: `DISPONIBLE`, `ALQUILADO`, `EN_MANTENIMIENTO`.

*   **Puertos (Interfaces):**
    *   `dominio/puerto/dao/MaquinariaDao.java` - Lectura:
        *   `List<Maquinaria> listarTodas()`
        *   `Optional<Maquinaria> buscarPorId(Long id)`
        *   `Optional<Maquinaria> buscarPorSerial(String serial)`
        *   `boolean existePorSerial(String serial)`
    *   `dominio/puerto/repositorio/MaquinariaRepositorio.java` - Escritura:
        *   `Maquinaria guardar(Maquinaria maquinaria)`
        *   `void eliminar(Long id)`

#### **2. Módulo Aplicación (`aplicacion/`)**

*   **CQRS - Comandos (Escritura):**
    *   `aplicacion/comando/ComandoCrearMaquinaria.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorCrearMaquinaria.java`:
        *   Validar serial único (usando `MaquinariaDao`).
        *   Validar tarifas positivas.
        *   Crear `Maquinaria` (dominio) con estado inicial `DISPONIBLE`.
        *   Persistir (usando `MaquinariaRepositorio`).
    *   `aplicacion/comando/ComandoActualizarMaquinaria.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorActualizarMaquinaria.java`:
        *   Buscar maquinaria existente.
        *   Validar serial único (si cambió).
        *   Actualizar campos (incluyendo estado si se proporciona).
        *   Persistir cambios.
    *   `aplicacion/comando/ComandoEliminarMaquinaria.java` - DTO inmutable.
    *   `aplicacion/comando/manejador/ManejadorEliminarMaquinaria.java`:
        *   Verificar que existe.
        *   Eliminar físicamente (usando `MaquinariaRepositorio`).

*   **CQRS - Consultas (Lectura):**
    *   `aplicacion/consulta/ConsultaListarMaquinaria.java` - DTO vacío.
    *   `aplicacion/consulta/manejador/ManejadorListarMaquinaria.java`:
        *   Obtener todas las maquinarias (usando `MaquinariaDao`).
        *   Retornar lista.
    *   `aplicacion/consulta/ConsultaBuscarMaquinaria.java` - DTO inmutable.
    *   `aplicacion/consulta/manejador/ManejadorBuscarMaquinaria.java`:
        *   Buscar por ID (usando `MaquinariaDao`).
        *   Retornar maquinaria o error si no existe.

#### **3. Módulo Infraestructura (`infraestructura/`)**

*   **Adaptadores de Persistencia:**
    *   `infraestructura/adaptador/entidad/MaquinariaEntity.java` - Entidad JPA.
    *   `infraestructura/adaptador/dao/JpaMaquinariaDao.java` - Implementa `MaquinariaDao`.
    *   `infraestructura/adaptador/repositorio/JpaMaquinariaRepositorio.java` - Implementa `MaquinariaRepositorio`.
    *   `infraestructura/adaptador/repositorio/JpaMaquinariaRepository.java` - Spring Data JPA interface.

*   **Controladores REST (Separados por CQRS):**
    *   `infraestructura/controlador/ComandoControladorMaquinaria.java`:
        *   `POST /api/maquimu/v1/maquinaria` → `ManejadorCrearMaquinaria`
        *   `PUT /api/maquimu/v1/maquinaria/{id}` → `ManejadorActualizarMaquinaria`
        *   `DELETE /api/maquimu/v1/maquinaria/{id}` → `ManejadorEliminarMaquinaria`
    *   `infraestructura/controlador/ConsultaControladorMaquinaria.java`:
        *   `GET /api/maquimu/v1/maquinaria` → `ManejadorListarMaquinaria`
        *   `GET /api/maquimu/v1/maquinaria/{id}` → `ManejadorBuscarMaquinaria`

### Especificaciones Técnicas

**API Endpoints:**
*   `GET /api/maquimu/v1/maquinaria` - Listar todas las maquinarias.
*   `GET /api/maquimu/v1/maquinaria/{id}` - Buscar maquinaria por ID.
*   `POST /api/maquimu/v1/maquinaria` - Crear nueva maquinaria (estado inicial: DISPONIBLE).
    *   Body: `{nombreEquipo, marca, modelo, serial, tarifaPorDia, tarifaPorHora, descripcion}`
    *   Response: `{id, nombreEquipo, marca, modelo, serial, estado: "DISPONIBLE", ...}`
*   `PUT /api/maquimu/v1/maquinaria/{id}` - Actualizar maquinaria existente (permite cambiar estado).
    *   Body: `{nombreEquipo, marca, modelo, serial, tarifaPorDia, tarifaPorHora, descripcion, estado}`
    *   Response: `{id, nombreEquipo, marca, modelo, serial, estado, ...}`
*   `DELETE /api/maquimu/v1/maquinaria/{id}` - Eliminar maquinaria (eliminación física).

**Base de Datos:**
*   Tabla `maquinaria` (ya existe en BD).
*   Constraint `UNIQUE` en columna `serial`.
*   No requiere migración adicional.

**Validaciones:**
*   Serial único (dominio + constraint BD).
*   Tarifas positivas (dominio).
*   Campos obligatorios: `nombreEquipo`, `marca`, `modelo`, `serial`, `tarifaPorDia`, `tarifaPorHora`.
*   Estado inicial: `DISPONIBLE` (automático en creación).
*   Estado editable: Permitido en actualización.

**Frontend (Angular):**
*   **Módulo:** `AdminModule`.
*   **Componentes:**
    *   `InventoryComponent`: Vista principal con tabla de listado.
    *   `MachineModalComponent`: Formulario para Crear/Editar (reutilizable).
*   **Servicios:** `MaquinariaService` (consume endpoints `/maquinaria`).
*   **Modelos:** Interfaz `Maquinaria`.
