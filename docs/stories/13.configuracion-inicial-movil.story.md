**Como:** Desarrollador Móvil
**Quiero:** Configurar el proyecto Android y implementar autenticación completa.
**Para:** Habilitar el desarrollo de funcionalidades móviles integradas con el backend y permitir que los usuarios inicien sesión desde la app.

**Criterios de Aceptación:**

**Configuración Base:**
*   El proyecto en `maquimu-mobile` compila y se ejecuta en el emulador/dispositivo.
*   La configuración de red (Retrofit) apunta a la IP local del backend.
*   La estructura del proyecto sigue MVVM con paquetes organizados.

**Autenticación (desde HU #03):**
*   Los usuarios pueden registrarse desde la app móvil (Cliente auto-registro).
*   Los usuarios pueden iniciar sesión con email y contraseña.
*   Si las credenciales son válidas, el usuario es redirigido a su dashboard.
*   Si las credenciales son incorrectas, se muestra un mensaje de error.
*   El token JWT se almacena en SharedPreferences.
*   El token se incluye automáticamente en todas las peticiones HTTP (Interceptor).

---

## Análisis Arquitectónico (Arquitecto)

### Decisiones de Diseño

**Patrón Arquitectónico:** MVVM (Model-View-ViewModel)
*   **Justificación:** Estándar de industria para Android. Desacopla la UI de la lógica de negocio y facilita el testing.
*   **Estructura de Paquetes:**
    `com.sena.proyecto`
    *   `data`: Capa de acceso a datos.
        *   `api`: Interfaces Retrofit (`ApiService`).
        *   `model`: Data Classes / DTOs.
        *   `repository`: Repositorios que abstraen la fuente de datos.
    *   `ui`: Capa de presentación (Activities, Fragments, ViewModels).
        *   `auth`: Login, Registro.
        *   `dashboard`: Pantalla principal.
    *   `utils`: Constantes, Extensiones, Helpers.

**Stack Tecnológico:**
*   **Lenguaje:** Kotlin
*   **Red:** Retrofit + Gson
*   **Concurrencia:** Kotlin Coroutines
*   **Inyección de Dependencias:** Manual (Container pattern simple) para evitar complejidad de Hilt en fase inicial.

### Especificaciones Técnicas

**Configuración de Red:**
*   **Base URL:** `http://10.0.2.2:8080/api/` (IP especial para emulador Android que apunta a localhost del host).
*   **Cliente HTTP:** OkHttp con logging interceptor (para debug).

**Refactorización Inicial:**
*   Mover `MainActivity` y `DashboardActivity` al paquete `ui`.
*   Crear paquete `data` vacío preparado para la integración.

### Validación Arquitectónica
**Validado por:** Usuario (Luis)

**Feedback incorporado:**
